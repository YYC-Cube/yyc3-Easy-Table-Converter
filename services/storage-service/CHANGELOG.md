## [2.0.0] - 2024-11-23

### 🎯 主要功能更新

#### 1. 多存储后端支持
- 实现了统一的存储适配器接口 (`IStorageAdapter`)
- 支持三种存储类型：本地存储、Amazon S3、Google Cloud Storage
- 通过适配器模式实现存储后端的无缝切换

#### 2. 存储适配器架构
- 创建了 `StorageAdapterFactory` 工厂类，支持动态创建不同类型的存储适配器
- 实现了 `LocalStorageAdapter`、`S3StorageAdapter` 和 `GCSStorageAdapter` 具体适配器
- 所有适配器共享相同的接口，确保功能一致性

#### 3. 增强的配置管理
- 支持通过环境变量灵活配置存储服务
- 添加了配置验证功能，确保配置有效性
- 提供默认配置和智能回退机制

#### 4. API 增强
- 更新了 `/health` 端点，提供更详细的服务状态信息
- 新增 `/config` 端点，用于获取存储服务配置信息
- 支持在运行时查看可用的存储类型

### 🔧 技术改进

#### 1. 代码架构优化
- 重构存储服务核心逻辑，采用更清晰的职责分离
- 移除硬编码的存储实现，转向基于接口的设计
- 提高代码可测试性和可维护性

#### 2. 扩展性提升
- 新的适配器架构使添加新的存储后端变得简单
- 每个存储适配器独立维护，便于单独更新和测试
- 支持同时配置多种存储后端，实现混合存储策略

#### 3. 错误处理增强
- 改进了错误处理逻辑，提供更详细的错误信息
- 添加配置验证，在服务启动时捕获潜在问题
- 实现优雅的降级机制，在存储类型不可用时自动切换

#### 4. 安全性改进
- S3 和 GCS 适配器支持临时签名URL，提高访问安全性
- 加强了文件路径验证，防止路径遍历攻击
- 支持细粒度的存储权限控制

### 📝 配置变更

#### 新增环境变量
- `STORAGE_TYPE`: 配置默认存储类型 (local/s3/gcs)
- `LOCAL_STORAGE_PATH`: 本地存储路径
- `S3_BUCKET_NAME`: S3 存储桶名称
- `S3_REGION`: S3 区域
- `S3_ACCESS_KEY_ID`: S3 访问密钥ID
- `S3_SECRET_ACCESS_KEY`: S3 密钥
- `S3_ENDPOINT`: 自定义 S3 端点 (可选)
- `S3_FORCE_PATH_STYLE`: 是否强制使用路径样式 (可选)
- `GCS_BUCKET_NAME`: GCS 存储桶名称
- `GCS_PROJECT_ID`: GCS 项目ID (可选)
- `GCS_KEY_FILENAME`: GCS 密钥文件路径 (可选)
- `MAX_FILE_SIZE`: 最大文件大小 (默认50MB)
- `FILE_EXPIRY_DAYS`: 文件过期天数 (默认7天)

### ⚠️ 迁移说明

1. **配置迁移**:
   - 创建 `.env` 文件，基于 `.env.example` 配置所需的环境变量
   - 对于本地存储，确保指定的存储路径存在且可写
   - 对于云存储，确保提供的凭证具有正确的访问权限

2. **数据迁移**:
   - 从旧版本升级时，存储在本地的文件需要手动迁移到新的存储位置
   - 对于迁移到云存储的情况，可以使用提供的存储适配器API批量上传文件

3. **API兼容性**:
   - 核心API接口保持不变，但内部实现已更新
   - 新增了配置验证和状态检查API，可用于监控服务健康状态

### 🚀 性能优化

### 🧪 测试增强
